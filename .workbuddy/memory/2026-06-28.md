### 核心功能
- `POST /start` — 发起审批（匹配模板、创建实例+第一个节点任务+通知）
- `POST /tasks/:id/approve` — 审批通过（节点完成→流转下一节点或最终通过）
- `POST /tasks/:id/reject` — 审批拒绝（流程直接结束为 rejected）
- `POST /tasks/:id/delegate` — 转审给他人
- `POST /withdraw/:instanceId` — 发起人撤回（仅 pending 且无已操作任务）
- `POST /cancel/:instanceId` — 管理员撤销
- 回调子系统：流程结束时 HTTP POST 通知 SCM/HRMS（axios + retry）
- 通知：站内消息（task_assigned / approved / rejected / completed / withdrawn）

## 出入库记录/盘点管理/多个列表页查询条件不生效修复

### 问题
- 选择筛选条件后，列表数据没有变化（筛选不生效）
- 影响模块：出入库记录、盘点管理、采购订单、收货单、采购计划、批次追溯、召回管理

### 根因
- `useEffect(() => { loadList(); }, [page, rowsPerPage])` 只监听 page 和 rowsPerPage
- 篮选条件不在依赖列表中，值变化不会触发 loadList
- onChange 里 `setPage(0)` 试图间接刷新，但如果 page 已经是0则不触发 useEffect
- 篮选条件（warehouseId/direction/movementType）不在依赖列表中，值变化不会触发 loadList
- onChange 里 `setPage(0)` 试图通过触发 page 变化间接刷新，但如果 page 已经是0则不会触发 useEffect

### 修复（7个文件）
- **StockMovementList** — 依赖 `[page, rowsPerPage, keyword, warehouseId, direction, movementType]`

## 产品损耗计算逻辑修复
- **问题1**（首次修复）：STOCK_TAKE_ADJUST 出库记录被计入 outboundQty → 排除 `movementType: { not: 'STOCK_TAKE_ADJUST' }`
- **问题2**（根因修复）：时间过滤导致所有历史出入库被排除。CostPriceRecord 的 endingQty=110（已含盘点调整），calculatedAt=15:55，之后无新出入库→expectedQty=actualQty=110→损耗=0
- **最终修复**：完全重写计算逻辑，去掉时间过滤，改用全量 aggregate：
  - 账面数量 = 总入库(全量) - 业务出库(全量,不含盘点调整)
  - 实存数量 = 当前库存qty（已含盘点调整）
  - 损耗 = 账面 - 实存
  - 鸡枞菌：300-180=120(账面) vs 110(实存) = 损耗10 ✅
