## 网关进程守护方案
- **问题根因**：所有服务（网关+4个后端）都是裸 `node xxx.js &` 后台进程，无进程守护。macOS休眠唤醒、终端会话关闭、内存压力等都可能导致进程意外死亡且不会自动重启。今晚已出现两次网关挂掉导致系统无法访问。
- **解决方案**：安装 pm2 进程管理器，所有服务由 pm2 守护，崩溃后自动重启
- **pm2 安装路径**：`/Users/pukun/.workbuddy/binaries/node/workspace/node_modules/pm2`
- **pm2 启动命令**：`NODE_PATH=/Users/pukun/.workbuddy/binaries/node/workspace/node_modules /Users/pukun/.workbuddy/binaries/node/versions/22.22.2/bin/node .../pm2/bin/pm2`
- **5个服务**：portal-server(4001), hrms-server(4002), scm-server(4003), mdm-server(4005), gateway(5174)
- **启动脚本**：`start-all-pm2.sh`（替代原来的 `start-all.sh`）
- **常用命令**：pm2 list / pm2 restart gateway / pm2 logs
