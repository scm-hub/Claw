## 修复销售订单审批提交后 Portal 工作台"我发起的"看不到申请
- **根因**：SCM 的 `globalId` 与 Portal/HRMS 不一致，导致 workflow-engine 存的 `submitterId` 和 Portal 查询用的 `globalId` 匹配不上
  - SCM Employee 的 `globalId` 是自动生成的 CUID（如 `cmr2xthjt000gnddoy9k0eexo`），而 HRMS 用的是 `g_emp001` 格式
  - SCM authenticate 中间件构建 `req.user` 时没包含 `globalId`
  - SCM JWT 也没包含 `globalId`
  - `submitterName` 也有 bug：代码取 `req.user.employee?.name`，但中间件没挂 `employee` 对象，导致存了空字符串
- **修复内容**：
  1. `auth.js`（SCM authenticate）：`req.user` 增加 `globalId`（优先取 JWT 中的 `decoded.globalId`，回退到 DB）和 `username`
  2. `auth.routes.js`（SCM SSO login）：SCM JWT payload 增加 `globalId`/`employeeName`/`email`（从 Portal SSO decoded 传递）
  3. `sales.routes.js`（submit-approval）：`submitterName` 改用 `req.user.name`（中间件已正确设置）
  4. `sync-from-hrms.js`：SQL SELECT 增加 `e.globalId`，`empData` 增加 `globalId` 字段，防止后续同步再丢
  5. **数据修复**：workflow-engine 的审批实例 `submitterId` 从 SCM employeeId 改为 `g_emp001`，`submitterName` 从空字符串改为"管理员"
  6. **数据修复**：SCM 全部 14 条 Employee 的 `globalId` 按 empNo 匹配更新为 HRMS 的值（g_emp001~g_emp014）
- **验证**：Portal 用 `g_emp001` 查询 workflow `my-flows` 能返回 2 条记录（采购订单 + 销售订单）
- **注意**：用户需要重新通过 Portal SSO 登录 SCM，才能拿到包含 `globalId` 的新 JWT

## 修复 workflow-engine 审批人解析为占位符导致"待我审批"看不到任务
- **根因**：`workflow.service.js` 的 `resolveNodeApprover()` 是纯占位符实现
  - `dept_manager` 类型存 `approverId = "dept_manager:<submitterId>"`，不是真实用户 ID
  - `role` 类型存 `approverId = "SALES_MANAGER"`（角色代码），也不是用户 ID
  - Portal "待我审批" 用 `globalId` 精确匹配 `approverId` → 永远匹配不上 → 无人能看到待审任务
- **修复**：
  1. 新建 `approverResolver.js`：用 mysql2 连接池查 HRMS DB（解析部门主管）和 Portal DB（解析角色审批人）
     - `resolveDeptManager(submitterId)`：globalId → Employee → Department.managerId → Manager Employee
     - `resolveRoleApprovers(approverRole)`：scmRole → Role → UserRole → PortalUser
     - dept_manager 找不到时回退到 approverRole 角色审批
  2. `workflow.service.js`：删除本地 `resolveNodeApprover`，改为 `import { resolveNodeApprover } from './approverResolver.js'`，两处调用加 `await`
  3. 安装 mysql2 依赖到 workflow-engine
  4. **数据修复**：已有 FlowTask 的 `approverId` 从 `"dept_manager:cmr2xthjt..."` 改为 `"g_emp008"`（苏建昌），`approverName` 从"部门主管"改为"苏建昌"，同时修复 FlowNotification 的 `recipientId`
- **当前审批人**：销售订单 SO20260702TRB1I4 的审批人是**苏建昌**（g_emp008），他是提交人管理员所在部门"杭州鲜当家生物科技有限公司"的主管
- **流程模板**：sales_order 模板只有一个节点"销售主管审批"，type=dept_manager, approverRole=SALES_MANAGER
