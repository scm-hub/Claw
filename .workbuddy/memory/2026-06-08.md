# 2026-06-08 工作日志

1. **部门表单必填字段** - 新建/编辑部门时，部门名称、上级部门、部门负责人均设为必填
   - 前端：三个字段加required标识，handleSubmit增加parentId和managerId的空值校验
   - 后端：POST和PUT路由增加parentId和managerId的必填校验
2. **部门导入功能** - 管理员可批量导入部门
   - 后端：`POST /api/departments/import` 上传Excel批量导入，智能列名识别，层级排序创建，负责人按姓名/工号匹配在职员工
   - 后端：`GET /api/departments/import-template` 下载导入模板
   - 前端：DepartmentImportDialog组件，支持拖拽上传、下载模板、导入结果展示（成功/跳过/错误详情）
3. **用户管理功能** - 解决HR和经理账号无法登录的问题
   - 根因：系统只有SUPER_ADMIN一个账号，没有用户管理功能创建HR/经理账号
   - 后端：`POST /api/auth/create-user` 管理员创建用户（指定角色），权限控制：只有SUPER_ADMIN可创建HR_ADMIN
   - 后端：`GET /api/auth/users` 获取用户列表，`PUT /api/auth/users/:id/role` 修改角色，`PUT /api/auth/users/:id/reset-password` 重置密码
   - 前端：UserManagement页面（/settings/users），包含创建用户、修改角色、重置密码功能
   - 侧边栏新增"系统设置"菜单（含"用户管理"子项），仅HR_ADMIN及以上角色可见
4. **部门显示顺序调整功能**
   - 数据库：Department模型新增sortOrder字段（Int, default 0），prisma db push同步
   - 后端：所有查询改用orderBy [{sortOrder:'asc'}, {name:'asc'}]，新增`PUT /api/departments/reorder`批量更新排序，`POST /api/departments/init-sort`初始化已有部门排序
   - 创建部门时自动设置sortOrder为同层级最大值
   - 前端：DepartmentRow每个行新增上移/下移按钮（ArrowUpward/ArrowDownward），点击后调用reorder API交换排序值
