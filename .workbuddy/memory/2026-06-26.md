## 产品管理保质期"0删不掉"Bug修复

- **需求**：产品管理新增产品时，保质期字段出现0删不掉的问题
- **根因**：`shelfLifeDays` 字段4处使用了 `|| 0` 模式 — (1)初始state=0 (2)value绑定`|| 0` (3)onChange强制Number (4)提交`Number() || 0`
- **修复**（按 numeric-zero-input-fix skill 四步方案）：
  1. 初始state：`shelfLifeDays: 0` → `shelfLifeDays: ''`
  2. value绑定：`form.shelfLifeDays || 0` → `form.shelfLifeDays ?? ''`
  3. onChange：`Number(e.target.value)` → `e.target.value === '' ? '' : Number(e.target.value)`
  4. 提交兜底：`Number(payload.shelfLifeDays) || 0` → `payload.shelfLifeDays === '' ? 0 : Number(payload.shelfLifeDays)`
  5. 新增 `placeholder="0"` + `onFocus={(e) => e.target.select()}` + `inputProps={{ min: 0 }}`
- **同类问题排查**：全局搜索 `|| 0` 发现 CustomerList(creditLimit)、ZoneLocationList(sortOrder/capacity)、StockMovementList(qty) 也有同类bug，待后续修复

## 成本价算法重写（加权平均法）（18:30）
- **旧算法**：12小时移动平均入库价 + 各类费用合计
- **新算法**：加权平均法（期初→入库→出库→结存→成本价）
  1. 期初数量=上次期末数量(无则0)，期初单价=上次期末单价(无则取Material.initialPurchasePrice)，期初金额=数量×单价
  2. 入库：上次计算时间→当前时间内的所有StockMovement(direction=IN)，单价通过PurchaseReceiptItem关联获取
  3. 销售成本价=(期初金额+入库金额)÷(期初数量+入库数量)（期初+入库一起加权平均）
  4. 出库：上次计算时间→当前时间内的所有StockMovement(direction=OUT)，金额=数量×销售成本价
  5. 结存数量=期初+入库-出库，结存金额=期初金额+入库金额-出库金额
  6. 加权平均价=结存金额÷结存数量，最终成本价=加权平均价+费用合计(feesTotal)
- **Schema改动**：CostPriceRecord 新增 beginningQty/beginningPrice/beginningAmount/inboundQty/inboundAmount/salesCostPrice/outboundQty/outboundAmount/endingQty/endingAmount/costPrice/periodStart/periodEnd；废弃 basePrice/totalCost/recordCount
- **StockMovement没有unitPrice字段**：入库单价需通过 refId→PurchaseReceiptItem.unitPrice 关联查询（批量查询receiptIds→receiptPriceMap）
- **前端成本价历史**：14列表格（期初数量/期初单价/期初金额/入库数量/入库金额/销售成本价/出库数量/出库金额/结存数量/成本价/计算时间）
- **展开明细**：计算摘要(11项flexWrap) + 入库明细子表格(8列) + 出库明细子表格(8列)
- **movements/avg-cost API**：改为直接取最新CostPriceRecord.costPrice，不再走旧的12小时平均逻辑
- **手动触发测试**：10个物料全部成功计算
