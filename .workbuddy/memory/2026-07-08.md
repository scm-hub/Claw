# 2026-07-08 工作日志

### Portal 角色映射修复
- 问题：吴月新分配了「HR管理员」，但登录 HRMS 后被推导为 SUPER_ADMIN，导致部门管理数据隔离不生效
- 根因：Portal `deriveHrmsRole()` 把"拥有全部 HRMS 模块"直接映射为 SUPER_ADMIN；且 `login()` 有安全网兜底，会按 HRMS 当前 role 强制覆盖为 SUPER_ADMIN
- 修复：
  - `getUserPermissions()` 改为优先使用 Portal `Role` 表上的显式 `hrmsRole` / `scmRole` 字段
  - 无显式映射时回退到按模块权限推导
  - 移除安全网兜底
  - 数据库补齐内置角色映射：HR管理员→HR_ADMIN、经理→MANAGER、普通员工→EMPLOYEE
- 吴月新 HRMS role 已手动修正为 HR_ADMIN
- 修复过程中引入 bug：误删了 `SSO_JWT_SECRET` / `SSO_JWT_EXPIRES_IN` 常量，导致登录报错 "SSO_JWT_SECRET is not defined"；已补回
- Portal server 已重启

### 部门管理数据隔离验证说明
- HR_ADMIN 按设计可看全部部门，不算隔离问题
- 若要测试部门负责人只看本部门：需要给用户分配「经理」角色，并确保是某部门的 `managerId`
