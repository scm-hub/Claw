## 修复：Portal 移动端布局配置页面「加载配置列表失败」

**问题**：综合平台点击「移动端布局」报 401。

**根因分析**：
1. `MobileLayoutConfig.jsx` 前端调用 `/api/scm/api/mobile-layout/configs` 路径拼接错误 → 修复为独立的 `scmApi` 实例（`baseURL: ''`），路径正确为 `/scm/api/mobile-layout/configs`
2. `/configs` 接口使用 SCM 的 `authenticate` 中间件，Portal 的 `sso_token` 中 `userId` 是 Portal DB 的用户 ID，SCM 查不到用户 → 401
3. `/config`（单个角色组查询）是公开接口，不需要认证 → 正常返回 200

**修复**：
- Portal 前端 `MobileLayoutConfig.jsx`：创建 `scmApi` axios 实例，附件 `sso_token`，baseURL 为空，直接请求 `/scm/api/...` 路径
- Portal 前端重新构建（`npm run build`）
- SCM 后端 `mobile-layout.routes.js`：管理接口（`/configs`、`/config/:roleGroup` PUT、`/config/:roleGroup/reset` POST）的认证从 `authenticate` 改为 `ssoAuth`（Portal SSO 中间件）
- 重启 gateway 和 scm-server
