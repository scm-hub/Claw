## 薪资计算页面滚动条修复（多轮排查）
**问题**：表格 20+ 列约 2500px 宽，但横向滚动条始终不出现，"保存全部"按钮被推到视口外
**根因**：flex column 布局中，滚动区域作为 flex 子元素默认 `min-width: auto`，不允许收缩到内容宽度以下 → 被表格撑到 2500px → 不溢出 → 无滚动条
**关键修复**：
1. `SalaryCalc.jsx` 根 div 加 `height: calc(100vh-112px); width: 100%; overflow: hidden`
2. 滚动区域 div 加 `min-width: 0; min-height: 0; flex: 1 1 0%; overflow: auto`（min-width:0 是核心！）
3. 表格用原生 `<table>` 替代 MUI `<Table>`（MUI 内部硬编码 width:100% 会覆盖 sx），设 `width: max-content`
4. 工具栏去掉 `flexGrow:1`，"批量操作"和"保存全部"按钮紧挨查询按钮放置
5. MainLayout 内容区加 `height: calc(100vh-64px); overflow: hidden`
**教训**：flex 子元素的 `min-width: auto` / `min-height: auto` 是滚动容器失效的头号杀手，必须显式设 0

### 最终方案（CSS Grid 替代 flex）
**问题**：上述 flex 方案在实际运行中仍不生效——多层 flex 嵌套下高度链断裂，滚动容器未被约束，内容溢出到整页
**根因**：flex 的 `flex: 1 1 0%` + `minHeight: 0` 在某些父容器组合下不可靠，Grid 的 `1fr` 更稳定
**最终修复**：
1. `SalaryCalc.jsx` 根容器从 `display: flex; flexDirection: column` 改为 `display: grid; gridTemplateRows: auto 1fr`
   - 第一行 `auto`：自适应内容高度（标题+工具栏+统计卡片）
   - 第二行 `1fr`：强制吃掉所有剩余空间，不会被子元素 content 撑开
2. 固定区域 div 去掉 `flexShrink: 0`（Grid 不需要）
3. 滚动容器去掉 `flex: 1 1 0%; minHeight: 0; minWidth: 0; position: relative`（Grid 自动分配高度）
4. 内层 `width: 2600px` + `minWidth: max-content` 保留，`table width: 100%` 填满内层 div
**验证**：蓝色调试边框确认容器边界正确，横向滚动条出现在表格底部（合计行下方）✅
**结论**：CSS Grid 的 `gridTemplateRows: auto 1fr` 比 flex 嵌套更可靠地实现"固定头部 + 滚动内容"布局，避免了 flex min-size 陷阱
