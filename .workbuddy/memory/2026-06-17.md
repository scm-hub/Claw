## MDM 完整同步字段缺失修复
- **问题**：用户在 HRMS 修改部门/员工数据后，点击 MDM「完整同步」，SCM 中员工职位(position)、部门上级(parentId)、负责人(managerId)、排序(sortOrder)等字段未同步
- **根因**：
  1. `mdm/server/src/services/scm-sync.js` 部门推送 SQL 只同步 name，缺失 parentId/managerId/sortOrder/status
  2. `scm-sync.js` 员工推送 SQL 只同步 name/empNo/email/phone/departmentId，缺失 position/status/hireDate
  3. `mdm/server/src/services/hrms-sync.js` 拉取部门时未拉取 managerId
  4. MDM `MasterDepartment` 模型缺少 managerHrmsId 字段
  5. 部门 parentId 需两趟处理（先建全部部门到 SCM 拿 DataMapping，再回填 parentId）
  6. 部门 managerId 需在员工同步后才能解析（通过 DataMapping 查 SCM 员工 ID）
- **修复文件**：
  - `mdm/server/prisma/schema.prisma`：MasterDepartment 加 `managerHrmsId String? @map("manager_hrms_id")`
  - `mdm/server/src/services/hrms-sync.js`：SQL 加 managerId，upsert data 加 managerHrmsId
  - `mdm/server/src/services/scm-sync.js`：重写，三步同步策略：
    1. Pass 1 部门基本字段（name/code/sortOrder/status）
    2. Pass 2 部门 parentId 回填（全部部门已建好后，通过 DataMapping 解析父部门 SCM ID）
    3. `pushEmployeesToScm` 补全 position/status/hireDate
    4. `updateDepartmentManagersInScm` 新函数：员工同步后解析 managerId（managerHrmsId → MasterEmployee → DataMapping → SCM 员工 ID）
    5. `pushAllToScm` 改为：部门 → 员工 → 部门负责人回填
- **验证结果**：5 部门全部字段一致（parentId/managerId/sortOrder/status），14 员工 position/status/hireDate 全部同步，HRMS 中 positionTitle 为 NULL 的 3 个员工在 SCM 中也为 NULL（数据一致）
