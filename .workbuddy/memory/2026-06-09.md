## 多方式登录 + 批量开通账号

### 需求
所有员工可通过工号/手机号/邮箱登录，默认密码 888888

### 后端变更
1. **auth.service.js** `login()` 改为 `identifier` 参数：
   - 先按邮箱查 User 表
   - 若未找到，按工号/手机号查 Employee，再关联 User
2. **auth.js** `/login` 接口：接受 `identifier`（兼容旧的 `email` 字段）
3. **auth.js** 新增 `POST /auth/batch-create-accounts`：
   - 为所有 `status=ACTIVE` 且无 User 的员工创建账号
   - 优先使用员工邮箱，冲突时用 `工号@hrms.internal`
   - 默认密码 888888（可通过请求体自定义）

### 前端变更
1. **Login.jsx**：登录框 label 改为"工号 / 手机号 / 邮箱"，移除 type="email"
2. **useAuth.js**：参数 `email` → `identifier`，发送 `{ identifier, password }`
3. **UserManagement.jsx**：添加「批量开通账号」按钮，调用 `/batch-create-accounts`

## 部门数据隔离（员工只看本部门数据）

### 需求
员工登录后只加载相同部门的数据，实现数据权限隔离

### 实现方案
- SUPER_ADMIN / HR_ADMIN → 看全部数据（不过滤）
- MANAGER / EMPLOYEE → 只看本部门数据

### 后端变更

1. **auth.service.js**：
   - JWT Token 新增 `departmentId` 字段
   - 登录返回新增 `departmentName`
   - 查询 Employee 时 include department 获取部门名称

2. **新建 middleware/departmentScope.js**：
   - 导出 `getDepartmentFilter(req)` 函数
   - 根据 `req.user.role` 和 `req.user.departmentId` 判断是否需要过滤
   - SUPER_ADMIN/HR_ADMIN 返回 null（不过滤），其他返回 departmentId

3. **路由添加部门过滤**（8个模块）：
   - employees.js: 员工列表+导出
   - dashboard.js: 仪表盘统计
   - leaves.js: 待审批请假
   - attendance.js: 考勤记录+导出
   - salary.js: 薪资记录+汇总
   - performance.js: 绩效列表
   - contracts.js: 合同列表
   - positions.js: 职位列表
   - recruitment.js: 招聘职位（Job.department 是字符串字段，需查部门名称）
   - reports.js: 报表

4. **服务层修改**：
   - dashboard.service.js: 所有 count 查询添加 departmentFilter 条件
   - leave.service.js: getPendingLeaves 按部门过滤
   - attendance.service.js: getAttendanceRecords 按部门过滤，include 返回部门名称
   - salary.service.js: getSalarySummary 按部门过滤
   - performance.service.js: listPerformances 按部门过滤
   - contract.service.js: listContracts 按部门过滤
   - recruitment.service.js: listJobs 按 department 字符串过滤

### 前端变更
- Dashboard.jsx: 当有部门过滤时显示蓝色提示栏"当前数据范围：XX部门"
- 部门分布图标题旁显示部门 Chip
