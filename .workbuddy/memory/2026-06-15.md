## 合同状态自动计算
- 需求：合同状态不可手动编辑，系统根据开始/结束日期自动判断
- 后端 `contract.service.js`:
  - 新增 `calcStatus(startDate, endDate, currentStatus)` 函数
    - RENEWED/TERMINATED 状态保持不变
    - 其他状态根据日期动态计算：startDate > now → PENDING, endDate < now → EXPIRED, 否则 ACTIVE

## 隐藏管理员角色员工

**需求**：把管理员从员工列表里隐藏

**实现方案**：
1. 修改 `server/src/services/employee.service.js`：
   - `listEmployees` 函数：增加 `NOT` 条件，排除 `user.role` 为 `SUPER_ADMIN` 或 `HR_ADMIN` 的员工
   - `exportEmployees` 函数：同样增加过滤条件

2. 过滤逻辑：
   ```javascript
   const where = {
     NOT: {
       user: {
         role: { in: ['SUPER_ADMIN', 'HR_ADMIN'] },
       },
     },
   };
   ```
   - 没有 User 账号的员工（user 为 null）不会被排除
   - 只有 `role` 为 `SUPER_ADMIN` 或 `HR_ADMIN` 的员工会被隐藏

**测试验证**：
- 数据库中原有 10 个员工（包括"管理员"SUPER_ADMIN 和"吴月新"HR_ADMIN）
- 过滤后显示 8 个员工
- 导出 Excel 也正确过滤了管理员

**重启后端**：修改后需重启后端服务使代码生效

## 修复合同部门筛选被角色权限强制覆盖

**问题**：王振东（MANAGER，部门=经理室）登录后，切换部门筛选无效，始终只显示经理室合同

**根因**：`contract.service.js` 中 `departmentFilter`（角色默认过滤）优先级高于用户选择的 `departmentId`

**修复**：
```js
// 修改前：departmentFilter 优先，departmentId 被忽略
if (departmentFilter && !employeeId) where.employee = { ...where.employee, departmentId: departmentFilter };
if (departmentId && !employeeId && !departmentFilter) where.employee = { ...where.employee, departmentId };

// 修改后：用户主动选择的 departmentId 优先
if (departmentId) {
  where.employee = { ...where.employee, departmentId };
} else if (departmentFilter && !employeeId) {
  where.employee = { ...where.employee, departmentId: departmentFilter };
}
```

**逻辑**：
- 用户选了部门 → 按选的部门过滤
- 用户没选部门 → 按角色权限过滤（MANAGER/EMPLOYEE 只看本部门）
- SUPER_ADMIN/HR_ADMIN → departmentFilter 为 null，看全部

**测试验证**：王振东账号 → 不选部门4条（经理室✅）→ 选商超事业部5条（商超事业部✅）
