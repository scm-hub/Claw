# 移动端布局可视化配置功能 — 2026-07-03 完成记录

## 做什么了
用户要求执行「C 完整版」方案：在 Portal 综合平台里加一个可视化配置页面，让管理员可以拖拽排序、勾选配置，实时预览移动端各角色的布局，保存后移动端 App 自动生效（无需重新打包）。

## 做了什么

### 1. 数据库（SCM DB）
- 新建表 `mobile_layout_configs`
  - roleGroup: 角色组（admin/warehouse/sales/purchase/finance/logistics/default）
  - navItems: JSON 底部导航配置
  - dashboardCards: JSON 仪表盘卡片配置
  - quickActions: JSON 快捷操作配置
- 写入 7 个角色组默认配置（与原来 navConfig.js 硬编码一致）

### 2. SCM 后端 API（/api/mobile-layout/...）
- GET /config?roleGroup=sales → 公开接口，移动端启动时拉取配置（无需登录）
- GET /presets → 预设图标和页面路由列表（给配置页面用）
- GET /configs → 所有角色组配置列表（需管理员登录）
- PUT /config/:roleGroup → 保存某角色组配置（需管理员登录）
- POST /config/:roleGroup/reset → 恢复默认配置（需管理员登录）
- 认证中间件改造：同时认 SCM JWT secret 和 Portal SSO secret（Portal 前端调用时不再 401）

### 3. Portal 前端配置页面
- 新建 `portal/client/src/pages/admin/MobileLayoutConfig.jsx`
- 功能：左侧角色组选择、中间可视化编辑（导航/卡片/快捷操作三项，支持上下移动排序、增删改、图标/路由下拉选择）、右侧实时手机模拟预览、保存/恢复默认
- 注册路由 `/admin/mobile-layout`
- MainLayout 菜单加入「移动端布局」（管理员可见）
- Portal vite 代理加 `/scm/api` → SCM 后端 4003

### 4. 移动端动态读取配置
- `authStore.js`：登录后自动拉取 layoutConfig 存入 store + localStorage
- `MobileLayout.jsx`：优先读 store 动态配置渲染底部导航，没有再 fallback 到 navConfig.js
- `Dashboard.jsx`：优先读 store 动态配置渲染仪表盘卡片和快捷操作，没有再 fallback
- 两个模式构建成功：gw（H5）和 native（Capacitor 原生壳）
- `npx cap sync` 完成，iOS/Android 原生项目已同步最新 Web 资源

- **根因**：`api.js` 的 `put` 方法对所有 body 执行 `JSON.stringify()`，FormData 被序列化为字符串而非 multipart
- **修复**：`client/src/lib/api.js` 第91行，`put` 方法增加 FormData 判断：
  ```js
  put: (url, body) => request(url, { method: 'PUT', body: body instanceof FormData ? body : JSON.stringify(body) }),
  ```
- **教训**：`post` 方法已有此判断，`put` 漏掉了（历史遗留）

- **需求**：成本价计算从"每个物料为一个计算单位"改为"每个物料的每种等级为一个计算单位"，没有等级就按物料计算
- **Prisma schema**：CostPriceRecord 新增 `gradeId` / `gradeName` 字段 + 复合索引 `[materialId, gradeId, calculatedAt]`
- **计算引擎重写** `cost-price.scheduler.js`：
  - 新增 `calculateOne(mat, gradeId, gradeName, now, feesTotal, feesDetail)` 独立函数
  - 通过 `prisma.materialGradeMapping.findMany` 获取物料关联等级
  - 始终包含无等级(null)兜底分组
  - 入库/出库查询根据 gradeId 过滤 StockMovement
  - 损耗查询不按 gradeId 过滤（StockTakeItem 无 gradeId 字段）
  - `getLatestCostPrice(materialId, gradeId)` 新增 gradeId 参数
- **前端** `PriceListPage.jsx`：成本价计算历史表格新增"等级"列，colSpan 15→16

### 2. 出入库记录列表增加单位字段
- **前端** `StockMovementList.jsx`：表头新增"单位"列，单位根据上游单据类型动态决定：
  - `SHIPPING_ORDER` / `SALES_OUTBOUND` → 销售单位 `salesUnit`
  - `PURCHASE_RECEIPT` → 采购单位 `purchaseUnit`
  - 其他 → 基准单位 `unit`
- **后端** `wms.routes.js`：出入库记录列表 API 的 material select 加入 `salesUnit` / `salesConversionFactor` / `purchaseUnit`

### 3. 销售订单/发货管理单位显示修正
- **销售订单** `sales.routes.js`：列表和详情 API material select 新增 `salesUnit` / `salesConversionFactor`；前端展开明细单位改为 `salesUnit`
- **发货管理** `logistics.routes.js`：明细 API 新增 `materialSalesUnit` 字段；前端 `ShippingOrderList.jsx` 单位改为 `salesUnit`

### 4. 出入库记录日期修改
- 将 SM20260703AHC857 和 SM202607034IR7BG 的出入库日期改为 2026/07/04 00:12
