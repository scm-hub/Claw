## 产品损耗计算历史字段调整
- **去掉**：加权平均价、损耗金额 两列
- **新增**：累计损耗率(cumulativeLossRate)，按月周期(29号→次月28号)计算
- **Prisma schema**：ProductLossRecord 新增 `cumulativeLossRate Decimal @default(0) @db.Decimal(5,2)`
- **后端算法**：损耗率与累计损耗率互换
  - 损耗率(lossRate) = 最新盘点记录 |diffQty| / bookQty × 100（单次盘点比率）
  - 累计损耗率(cumulativeLossRate) = 全量损耗数量 / 账面数量 × 100（全量累计比率）
  - 直接从 StockTakeItem 查询最新一条已完成盘点记录，取 bookQty 和 diffQty
  - 例：鸡枞菌 最新盘点 diffQty=-2, bookQty=110 → 损耗率=1.82%；全量 lossQty=12, expectedQty=120 → 累计损耗率=10%
- **后端**：detail API 返回增加 cumulativeLossRate；recalculate 存储 periodStart/periodEnd 为月周期日期；create 数据不再依赖 lossAmount/weightedAvgPrice 变量（设为0）
- **前端**：表头和数据行去掉加权平均价+损耗金额，新增累计损耗率列(带颜色：>5%红色、>0%橙色)；合计行去掉损耗金额；展开明细摘要去掉加权平均价+损耗金额，新增累计损耗率

## 费用登记重新设计：按物料指定三类费用
- **旧设计**：费用登记是通用费用列表（feeName + amount + isActive），不关联物料
- **新设计**：每物料一行，列 = 物料 | 包装费用 | 单盒成本 | 采购到货运费/盒 | 费用合计
- **Prisma schema**：FeeRecord 新增 `materialId String?` 关联 Material，新增 feeType PACKAGING/BOX_COST/FREIGHT；Material 加反向关系 feeRecords
- **后端新增 API**：
  - `GET /sales/fees/by-material`：查所有 ACTIVE 物料 + 对应三类费用，组装成每物料一行
  - `PUT /sales/fees/by-material/:materialId`：批量保存某物料的三类费用（有则更新，无则创建）
- **`/cost-price/latest` 修改**：费用合计改为按物料查三类费用(PACKAGING+BOX_COST+FREIGHT)而非全局所有启用费用
- **前端 FeeRegistrationTab 重写**：改为表格每物料一行，点击编辑按钮进入 inline 编辑模式（三个输入框），合计实时计算，底部合计行
