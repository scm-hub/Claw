# 2026-06-13 工作记录

## HRMS - 请假时长支持0.5天和按小时
- Prisma schema：Leave 模型新增 `duration` (Float, default 1)、`durationUnit` (String, default "DAY")、`startTime` (String?, 按小时请假时的起始时间)
- 后端 leave.service.js：applyLeave/resubmitLeave 支持 duration/durationUnit/startTime 字段
- 后端 getLeaveBalance：使用 duration/durationUnit 计算已用天数（0.5天直接使用，小时/8=天数），旧数据兼容按日期差计算
- 后端 routes/leaves.js：resubmit 接口增加 duration/durationUnit/startTime 参数传递
- 前端 LeaveApply.jsx：
  - 新增"时长类型"选择器（按天/按小时）
  - 按天：下拉选择0.5/1/1.5/2/3/5/7/10/14/15/30天
  - 按小时：数字输入1-24小时 + 开始时间选择器
  - 时长预览提示（显示请假时长和起始时间）
  - 自动计算结束日期
- 前端 LeaveBalance.jsx：
  - calcDays 改为使用 duration/durationUnit（兼容旧数据）
  - formatDuration 函数显示格式化时长（"0.5天（半天）"、"2小时（14:00起）"）
  - 余额已用天数显示为1位小数
  - 时长列用 Chip 展示，按天=primary色，按小时=secondary色
  - ResubmitDialog 同步支持时长类型选择
- 前端 LeaveApproval.jsx：
  - calcDays/formatDuration 同步更新
  - 时长列改用 Chip 展示
  - 审批弹窗显示格式化时长

## HRMS - 合同续签功能
- Prisma schema：Contract 模型新增 `renewedFromId` (String?, @unique) 和自关联关系 `ContractRenewal`
  - renewedFrom: 指向原合同（新合同的字段）
  - renewedTo: 指向续签后的新合同（原合同的字段）
- 后端 contract.service.js：
  - renewContract(id, data)：校验原合同存在+未续签→原合同标记EXPIRED→创建新合同并关联
  - getRenewalHistory(id)：沿续签链向前和向后查找，返回完整续签链（从旧到新排列）
  - listContracts/getContract 查询 include renewedFrom 和 renewedTo
- 后端 contracts.js 路由：
  - POST /:id/renew：续签合同
  - GET /:id/renewal-history：获取续签历史链
- 前端 ContractList.jsx：
  - RenewDialog 续签弹窗：显示原合同信息，默认开始日期=原合同结束日+1天，默认期限=原合同等长
  - RenewalHistoryDialog 续签历史弹窗：Stepper 组件展示续签链，标记"当前"合同
  - 表格状态列：已续签→"已续签"Chip，续签自→"续签"Chip
  - 操作列：续签按钮（即将到期/已过期且未被续签）、续签历史按钮（有续签关联时）
  - 统计卡片：新增"已续签"计数
  - 已过期未续签的行淡红底色提示
