### 修复 admin 500 报错和 HRMS globalId 缺失 18:25
- 根因1：admin 的 HRMS User 表中 employeeId 指向已被删除的 Employee，导致 SSO login 时 user.employee=null，portalUser.upsert 把 employeeNo/departmentName 全置空
- 根因2：admin 的 HRMS User.role='EMPLOYEE'，Portal requireAdmin 中间件只检查顶层 role 字段，导致 403
- 根因3：HRMS User.globalId 字段不存在（schema 里有但 db push 没执行过），prisma user.findUnique 报 "globalId expected non-nullable"
- 根因4：HRMS Employee.globalId 存在 NULL 值，db push 拒绝执行

修复：
1. HRMS Employee XDJ000000 '超级管理员' 杭州鲜当家 createdAt 入职
2. UPDATE HRMS User admin employeeId, role='SUPER_ADMIN'
3. UPDATE HRMS Employee SET globalId=CONCAT('EMP_', employeeNo) WHERE globalId IS NULL
4. db push 同步 HRMS User.globalId 字段
5. Portal requireAdmin 中间件增加 systemRoles 兼容判断

教训：
- SSO 用户的 Employee 记录不能删除，否则 login 流程会破坏缓存表
- 涉及 prisma schema 必填字段的表，db push 必须先补 NULL
- requireAdmin 应该兼容 systemRoles 不只是顶层 role

验证：admin 登录后 portal_user.employeeNo='XDJ000000', name='超级管理员', departmentName='杭州鲜当家', /api/users 返回 99 条
