# 2026-06-03 工作日志

## HRMS 系统维护与修复

1. **启动后端服务** - MySQL正常运行，Prisma Client生成+db push同步，服务启动在端口4002
2. **菜单顺序调整** - 将「部门管理」移到「员工管理」上方（MainLayout.jsx menuItems数组）
3. **部门负责人选择Bug修复** - 根因：DepartmentTree.jsx第216行 `empRes.items` 取错了字段路径。后端员工列表API返回格式为 `{ success, data: { page, pageSize, total, data: [...] } }`，axios拦截器解包后为 `{ success, data: { ... } }`，员工数组在 `empRes.data.data` 而非 `empRes.items`。修复为 `empRes.data?.data || empRes.data || []`
4. **端口说明** - 向用户解释从5173(Vite开发模式)切换到4002(生产构建模式)的原因：Vite代理不稳定、端口冲突、"出错了"页面问题
5. **部门负责人下拉框改为模糊搜索** - 将TextField select替换为MUI Autocomplete组件，支持按姓名/工号/职位/部门名模糊搜索，显示两行信息（姓名工号职位+所属部门）
6. **部门管理添加导出功能** - 后端新增 `GET /api/departments/export` 路由，导出包含两个Sheet的Excel：①部门列表（名称/上级/负责人/人数/创建时间）②部门员工明细（每个部门下所有员工信息）。前端添加「导出Excel」按钮
7. **员工状态变更报错修复** - Bug：编辑员工时将在职改为停职报 Prisma DateTime 错误。根因：前端发送空字符串日期字段（birthday/hireDate/leaveDate），Prisma无法将空字符串解析为DateTime。修复：`employee.service.js` 的 `updateEmployee` 和 `createEmployee` 中，对日期字段空字符串转null处理。服务端口从4002迁移到4003（旧进程无法kill）
8. **员工管理添加按状态筛选** - 在部门筛选旁新增状态下拉筛选（在职/停职/离职），选中后显示对应颜色Chip可清除，导出也带状态条件
9. **身份证号自动解析** - EmployeeForm.jsx新增handleIdCardChange，输入18位身份证号后自动：①提取第7-14位为出生日期填入birthday字段 ②第17位奇数=男/偶数=女自动填入gender ③出生日期下方显示年龄（实时计算）。身份证号格式不正确时显示提示
