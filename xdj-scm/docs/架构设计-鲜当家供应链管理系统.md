## 一、系统总体架构

### 1.1 架构风格

采用 **单体模块化 + 预留金蝶适配层** 架构，核心业务自研，金蝶对接预留接口：

### 1.2 技术选型

| 层 | 技术 | 理由 |
|---|------|------|
| **前端PC** | React 18 + Vite + MUI + Tailwind | 与HRMS技术栈统一，团队熟悉 |
| **前端H5** | React 18 + Vite + Ant Design Mobile | 移动端组件丰富，与PC端技术栈统一 |
| **API Gateway** | Nginx | 路由、限流、认证统一入口 |
| **后端** | Node.js 22 + Express + Prisma | 与HRMS统一，Prisma ORM高效 |
| **数据库** | MySQL 8.0 (xdj_scm_db) | **独立数据库**，与SCM/HRMS隔离 |
| **缓存** | Redis 7 | 库存查询缓存、价目表缓存、会话管理、温度数据缓存 |
| **消息队列** | BullMQ (Redis-backed) | 采购入库事件、审批流事件、预警事件驱动 |
| **实时通信** | WebSocket (Socket.io) | 冷链温度实时推送、库存变更实时通知 |
| **文件存储** | 本地FS / MinIO | 发货单、退货凭证、合同附件等 |
| **部署** | Docker Compose | 开发/测试环境一键编排 |

## 二、模块拆分与职责

### 2.1 模块清单（21个模块）

| # | 模块 | 职责 | 核心实体 | 优先级 |
|---|------|------|----------|--------|
| 1 | **auth** | 认证、RBAC、多数据中心切换 | User, Role, DataCenter | P0 |
| 2 | **master** | 基础数据（产品/客户/供应商/仓库/部门/员工） | Material, Customer, Supplier, Warehouse, Department, Employee | P0 |
| 3 | **purchase** | 采购计划、采购订单、入库确认 | PurchasePlan, PurchaseOrder, PurchaseReceipt | P0 |
| 4 | **sales** | 销售计划、销售订单、客户、价目表 | SalesPlan, SalesOrder, Customer, PriceList | P0 |
| 5 | **inventory** | 库存查询、锁定/释放、库存预警 | Inventory, StockLock | P0 |
| 6 | **cost** | 标准成本计算、成本配置、成本快照 | StandardCost, CostConfig, CostSnapshot | P0 |
| 7 | **wms** | 仓储管理：库位、出入库操作、盘点、FIFO/FEFO | WarehouseZone, WarehouseLocation, StockMovement, StockTake | P0 |
| 8 | **traceability** | 批次追溯：批次编码、正反向追溯、召回 | Batch, BatchTracking, RecallOrder | P0 |
| 9 | **finance** | 应收应付、发票、对账、信用管控 | AccountReceivable, AccountPayable, Invoice, Payment | P0 |
| 10 | **approval** | 单级审批流（特价+售后+采购计划+销售计划） | ApprovalFlow | P0 |
| 11 | **shipping** | 发货管理、运输费用跟踪 | ShippingOrder | P1 |
| 12 | **logistics** | 物流商管理、运单、配送路线、运费结算 | LogisticsProvider, Waybill, DeliveryRoute, FreightSettlement | P1 |
| 13 | **coldchain** | 冷链温度监控、超标预警、合规报告 | Sensor, TemperatureRecord, TemperatureAlert | P1 |
| 14 | **barcode** | 条码生成、扫码作业、批次扫码 | BarcodeRecord | P1 |
| 15 | **aftersales** | 售后申请、退货收货、退款 | AfterSalesRecord | P1 |
| 16 | **contract** | 采购/销售合同管理、到期提醒 | Contract | P1 |
| 17 | **notification** | 通知仓库备货、审批消息、预警推送 | Notification | P1 |
| 18 | **analytics** | 数据分析看板、多维报表 | (视图聚合，少量配置表) | P2 |
| 19 | **alert** | 预警规则、预警记录、集中展示 | AlertRule, AlertRecord | P2 |
| 20 | **supplier-eval** | 供应商准入、评估评分 | SupplierEvaluation | P2 |
| 21 | **kingdee** | 金蝶API封装（预留，P2启用） | SyncLog, MappingConfig | P2 |

```sql
    id VARCHAR(20) PRIMARY KEY,

    password_hash VARCHAR(255) NOT NULL,
    employee_id VARCHAR(20),
    role VARCHAR(30) NOT NULL DEFAULT 'EMPLOYEE',
```

```sql
    error_message TEXT,
    retry_count INT DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_ksl_type_status (sync_type, status),
    INDEX idx_ksl_date (created_at)
) ENGINE=InnoDB;
```

### 3.11 状态机汇总

```
采购计划：  DRAFT → PENDING_APPROVAL → APPROVED → PUBLISHED → EXECUTING → COMPLETED/CLOSED
                                    → REJECTED → DRAFT

销售计划：  DRAFT → PENDING_APPROVAL → APPROVED → EXECUTING → COMPLETED
                                    → REJECTED → DRAFT

采购订单：  PENDING → CONFIRMED → PARTIAL → COMPLETED / CANCELLED
销售订单：  DRAFT → PENDING_APPROVAL → APPROVED → LOCKED → SHIPPING → COMPLETED / CANCELLED
发货单：    PENDING → PICKING → PACKING → SHIPPED → DELIVERED / RETURNED
售后单：    PENDING → APPROVED → RECEIVED → COMPLETED / REJECTED
盘点单：    DRAFT → IN_PROGRESS → COMPLETED / CANCELLED
批次：      ACTIVE → DEPLETED / EXPIRED / RECALLED
应收/应付：  PENDING → PARTIAL → SETTLED / OVERDUE
召回单：    PENDING → NOTIFIED → IN_PROGRESS → COMPLETED / CANCELLED
运单：      CREATED → DISPATCHED → IN_TRANSIT → DELIVERED / RETURNED
温度告警：  ACTIVE → ACKNOWLEDGED → RESOLVED
预警记录：  ACTIVE → ACKNOWLEDGED → RESOLVED
```

## 五、标准成本价格引擎

### 5.1 计算公式

```
标准成本（不含税）= 采购成本 + 采购运输分摊成本 + 包装成本 + 运输成本

其中：
  采购成本 = MIN(最新入库价, 档案采购价)  ← 不含税
  采购运输分摊 = 上月采购费用分配单价 OR 预定义值  ← 不含税
  包装成本 = 预定义值  ← 不含税
  运输成本 = 预定义值  ← 不含税
```

### 5.2 取值优先级链

```
采购成本：
  1. 最新入库价 (materials.latest_receipt_price) → 不含税
  2. 档案定义采购价 (materials.archive_purchase_price) → 不含税
  3. 两者取较低者

采购运输分摊：
  1. 上月采购费用分配单价 (cost_configs: key=TRANSPORT_ALLOC_{materialId})
  2. 预定义分配值 (cost_configs: key=DEFAULT_TRANSPORT_ALLOC)

包装成本：
  1. 预定义价格 (cost_configs: key=PACKAGING_COST_{materialId})
  2. 通用默认值 (cost_configs: key=DEFAULT_PACKAGING_COST)

运输成本：
  1. 预定义价格 (cost_configs: key=TRANSPORT_COST_{materialId})
  2. 通用默认值 (cost_configs: key=DEFAULT_TRANSPORT_COST)
```

### 5.3 计算触发时机

| 触发 | 条件 | 说明 |
|------|------|------|
| **入库事件** | 采购入库单确认后 | 更新latest_receipt_price，触发该产品重算 |
| **每日定时** | 凌晨3:00 | 全量重算所有ACTIVE产品的标准成本 |
| **配置变更** | cost_configs修改时 | 触发受影响产品的重算 |
| **手动触发** | 管理员点击「重算成本」 | 选定产品或全量重算 |
