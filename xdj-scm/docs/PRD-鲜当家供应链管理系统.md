# 鲜当家食用菌供应链管理系统 — 产品需求文档 (PRD)

> **版本**：V3.0 | **日期**：2026-06-17 | **负责人**：许清楚（产品经理）  
> **适用范围**：杭州鲜当家生物科技 — 食用菌全品类供应链管理  
> **V3.0 变更**：新增10个模块（WMS/批次追溯/财务结算/物流管理/冷链温控/扫码作业/合同管理/供应商评估/数据分析/预警中心），移动端新增销售计划/销售订单/实时库存功能  
> **V2.1 变更**：项目更名为 xdj-scm  
> **V2.0 变更**：系统完全独立于现有SCM

---

## 一、产品目标

| 维度 | 目标 |
|------|------|
| **业务贯通** | 采购计划→采购入库→仓储管理→定价→销售→物流发货→售后→财务结算全链路数字化 |
| **食品安全** | 批次追溯全链路可查，从供应商到客户全程可追溯，支持快速召回 |
| **移动赋能** | 销售人员随时随地完成销售计划、下单、查库存、发起售后 |
| **成本透明** | 自动计算产品标准成本（采购+包装+运输），为定价提供数据支撑 |
| **冷链保障** | 冷链仓储和运输温度实时监控，异常预警，保障食用菌品质 |
| **财务闭环** | 应收应付自动生成，发票管理、对账、信用管控一体化 |
| **智能预警** | 库存不足、保质期临近、价格异常、信用超限等多维预警 |
| **金蝶预留** | 预留金蝶对接接口层，金蝶部署后可快速启用双向同步 |
| **系统独立** | 与现有SCM系统完全隔离，独立数据库(xdj_scm_db)、独立部署、独立运维 |

## 二、用户故事

### US-1 采购计划与采购管理（采购部）
> 作为采购员，我先制定采购计划（按月/季/年），经审批后下达执行，再根据计划创建采购订单，供应商确认后生成入库单，入库后自动更新库存和最新入库价。计划执行进度可实时跟踪。

### US-2 移动端销售计划（销售员/销售经理）
> 作为销售员，我在手机上制定和管理销售计划（按月/季），设定目标客户和预估销量，销售经理可审批销售计划，跟踪实际完成情况。

### US-3 移动端销售订单（销售员）
> 作为销售员，我在手机上创建销售订单（选客户和产品、自动取价、锁定库存），查看历史订单状态，特价走审批流程。下单后仓库立刻收到备货通知。

### US-4 移动端实时库存（销售员）
> 作为销售员，我在手机上实时查看各仓库各产品的可用库存数量，下单时自动检查库存是否充足，避免超卖。

### US-5 销售定价（销售经理）
> 作为销售经理，我维护不同客户/产品的价目表，设置最低限价和阶梯定价，批量调价一键生效。

### US-6 标准成本计算（财务/管理层）
> 作为管理者，我查看产品的标准成本（采购成本+包装+运输），以此作为定价决策依据，成本自动按规则更新。

### US-7 仓储管理WMS（仓库管理员）
> 作为仓库管理员，我管理库区→货架→库位三级结构，执行收货→质检→上架→拣货→复核→打包→发货全流程操作，定期盘点并处理盘盈盘亏。食用菌产品严格执行先进先出/先到期先出(FIFO/FEFO)。

### US-8 批次追溯（质量/管理层）
> 作为质量管理员，我从供应商批次→入库→出库→发货→客户全链路追溯产品流向。正向追溯查某批次卖给了谁，反向追溯查某客户订单的货来自哪个供应商。发现质量问题可快速锁定批次范围并发起召回。

### US-9 财务结算（财务部）
> 作为财务人员，销售发货后自动生成应收账款，采购入库后自动生成应付账款，我管理开票、收款/付款核销、与客户/供应商定期对账。客户信用超限时系统自动拦截或预警。

### US-10 物流管理（物流/仓储）
> 作为物流人员，我管理物流商和运单，关联发货单跟踪配送状态，支持多客户合并配送和路线优化，按物流商月结对账并分摊运费到订单。

### US-11 冷链温控监控（质量/IT）
> 作为质量管理员，我实时监控冷库和冷链运输的温度数据，温度超标时自动推送告警，可导出历史温度曲线和合规报告。

### US-12 发货协同（销售+仓库+物流）
> 作为销售员，我维护客户收货地址，发货通知自动携带地址；物流费用通过应付单跟踪，可按合同履约分摊成本。

### US-13 售后处理（销售员+仓库+财务）
> 作为销售员，我在手机上发起退货申请（退款退货/退款不退货/补货），仓库收货后系统自动生成负数应收单，财务走退款流程。

### US-14 扫码作业（仓库/生产）
> 作为仓库操作员，我用PDA或手机扫码完成收货、上架、拣货、发货等操作，扫码即关联批次信息，实现无纸化作业。

### US-15 合同管理（采购/销售/法务）
> 作为采购/销售人员，我管理采购合同和销售合同的签订、执行跟踪、到期提醒，合同关联对应的采购订单/销售订单。

### US-16 供应商评估（采购/管理层）
> 作为采购经理，我对供应商进行准入审核和定期评估，跟踪交货准时率、质量合格率、价格竞争力，供应商评分自动汇总。

### US-17 数据分析与报表（管理层）
> 作为管理者，我通过数据看板查看销售额趋势、毛利分析、库存周转率、采购价格波动、客户贡献度等多维分析，支持决策。

### US-18 预警中心（全员）
> 作为相关角色，我收到系统自动推送的预警：库存不足、保质期临近、价格异常、客户信用超限、采购计划超期、冷链温度异常等。

### US-19 多数据中心切换（IT管理员）
> 作为IT管理员，我在移动端启动时可切换数据中心（如杭州/外地仓），访问对应数据。

## 四、业务流程图

### 4.1 采购计划与入库流程

```
采购员编制采购计划 → 提交审批（采购主管/总经理单级审批）
                              ↓
                    ┌─ 审批通过 → 计划下达 → 可关联创建采购订单
                    └─ 审批驳回 → 修改重新提交
                              ↓
创建采购订单（关联采购计划）→ 供应商确认 → 仓库入库确认（WMS收货+质检+上架）
                                              ↓
                                    自动生成批次号 + 更新库存 + 更新最新入库价
                                              ↓
                                    更新采购计划执行进度（已采购数量）
                                              ↓
                                    自动生成应付账款 → 触发标准成本重算
```

> 📌 金蝶对接预留：入库确认后触发 `onPurchaseReceipt` 钩子，金蝶启用时在此推送数据

### 4.2 移动端销售流程

```
销售员打开手机App → 查看销售计划 → 查看实时库存
                                    ↓
选择客户 → 选择产品 → 系统自动取价（校验库存是否充足）
                                    ↓
                        ┌─ 价格≥限价 → 直接下单 → 锁定库存 → 通知仓库备货
                        └─ 价格<限价 → 提交特价审批(销售经理单级审批)
                                                    ↓
                                          ┌─ 审批通过 → 下单 → 锁定库存 → 通知仓库
                                          └─ 审批驳回 → 修改价格或取消
```

### 4.3 仓储出入库流程（WMS）

```
入库：采购到货 → 扫码收货 → 质检 → 分配库位 → 上架 → 库存更新 → 批次记录
出库：销售订单 → 拣货单生成 → 扫码拣货(FIFO/FEFO) → 复核 → 打包 → 发货
盘点：选择范围 → 扫码盘点 → 系统对比 → 差异处理 → 审核确认 → 库存调整
```

### 4.4 批次追溯流程

```
正向追溯（供应商→客户）：
  选择采购批次 → 查看入库记录 → 查看出库记录 → 查看关联销售订单 → 查看发货客户

反向追溯（客户→供应商）：
  选择客户订单 → 查看关联批次 → 查看入库记录 → 查看供应商

召回流程：
  锁定问题批次 → 系统自动列出所有关联客户 → 发起召回通知 → 退货处理
```

### 4.5 标准成本计算流程

```
触发计算 → 取采购成本（优先最新入库价 → 其次档案采购价）
                ↓
        + 采购运输成本（优先上月分配单价 → 其次预定义值）
                ↓
        + 包装成本（预定义值）
                ↓
        + 运输成本（预定义值）
                ↓
        = 标准成本（不含税）→ 存入产品标准成本表 → 供定价参考
```

### 4.6 财务结算流程

```
销售发货 → 自动生成应收账款 → 开票 → 客户付款 → 收款核销
采购入库 → 自动生成应付账款 → 供应商对账 → 付款 → 付款核销
售后退货 → 生成负数应收 → 审核后退款
信用管控 → 下单时校验信用额度 → 超限拦截/预警
```

### 4.7 物流发货流程

```
销售订单确认 → 创建发货单（携带收货地址）→ 仓库拣货打包
                    ↓
          选择物流商 → 创建运单 → 分配配送路线
                    ↓
          物流配送 → 运单状态跟踪 → 客户签收
                    ↓
          月结对账 → 运费分摊到订单 → 生成应付
```

### 4.8 冷链监控流程

```
温湿度传感器 → 数据采集(每5分钟) → 写入温度记录
                    ↓
          实时看板展示 → 超阈值 → 自动推送告警（App+短信）
                    ↓
          历史温度曲线 → 合规报告导出
```

### 4.9 售后流程

```
销售员发起申请 → 选择类型(退款退货/退款不退货/补货)
                            ↓
            ┌─ 涉及退货 → 审批通过 → 仓库收货（扫码） → 系统生成负数应收单
            └─ 不涉及退货 → 审批通过 → 自动生成负数应收单冲销
                            ↓
                    负数应收审核 → 财务退款流程
```

## 五、已确认决策

| # | 决策 | 结论 | 影响 |
|---|------|------|------|
| 1 | 金蝶系统 | **尚未部署，预留接口后续启用** | Phase 10 金蝶对接后置为 P2 |
| 2 | 采购系统 | **不用晓豆，直接在本系统内开发** | 含采购计划+采购订单+入库+供应商管理 |
| 3 | 特价审批 | **单层级审批（销售经理）** | 审批流简化，不做多级流转 |
| 4 | 采购计划 | **纳入采购管理模块** | 计划编制+审批+下达+执行跟踪 |
| 5 | **系统独立性** | **与现有SCM完全无关** | 独立数据库xdj_scm_db、独立项目xdj-scm、独立部署 |
| 6 | **功能补全** | **新增10个模块+移动端3功能** | 系统从11模块扩展到21模块，覆盖完整供应链 |
| 7 | **移动端销售** | **增加销售计划、销售订单、实时库存** | 销售人员全移动端作业 |
| 8 | **WMS+批次追溯** | **P0级别，与采购同期开发** | 食品安全底线，数据模型从开始就设计 |
| 9 | **财务结算** | **P0级别，与销售同期开发** | 应收应付自动生成，业务闭环 |
| 10 | **冷链监控** | **P1级别，仓储先行** | 温度传感器对接需现场调研 |
