## 一、产品概述

### 1.1 产品定位
一套面向中小型企业的一站式人力资源管理系统，覆盖员工管理、考勤、薪酬、招聘、绩效等核心 HR 场景，帮助企业实现人力资源数字化管理。

### 1.2 产品目标
| 目标 | 描述 | 衡量指标 |
|------|------|---------|
| 提升效率 | 替代手工 Excel 管理，减少 80% 重复操作 | HR 日常操作时间减少 |
| 数据透明 | 员工自助查询个人信息、薪资、假期 | 员工自助使用率 > 60% |
| 合规管理 | 自动计算薪资、个税、社保 | 计算错误率 < 0.1% |
| 决策支持 | 提供人员流动、成本分析报表 | 管理层月度报表使用率 > 80% |

### 1.3 目标用户
| 角色 | 核心诉求 |
|------|---------|
| HR 管理员 | 员工信息管理、考勤统计、薪资核算、招聘管理 |
| 部门经理 | 团队人员查看、绩效评估、审批流程 |
| 普通员工 | 个人信息查看、考勤打卡、请假申请、薪资查询 |
| 系统管理员 | 权限配置、系统设置、数据备份 |

## 二、用户故事

### P0 核心故事
1. **US-001**：作为 HR，我希望能够录入/编辑/查看员工信息，以便管理员工档案
2. **US-002**：作为 HR，我希望能够批量导入员工信息，以便快速完成新入职批次录入
3. **US-003**：作为员工，我希望能够在线打卡，以便记录我的考勤
4. **US-004**：作为 HR，我希望能够查看/统计全员考勤数据，以便掌握出勤情况
5. **US-005**：作为员工，我希望能够在线提交请假申请，以便规范化请假流程
6. **US-006**：作为经理，我希望能够审批下属的请假申请，以便管控团队出勤
7. **US-007**：作为 HR，我希望能够自动计算月度薪资（含社保、个税），以便准确发放工资
8. **US-008**：作为员工，我希望能够查看自己的薪资明细，以便了解收入构成
9. **US-009**：作为 HR，我希望能够发布招聘职位并管理候选人，以便高效招聘
10. **US-010**：作为系统管理员，我希望能够管理用户角色和权限，以便保障数据安全

### P1 重要故事
11. **US-011**：作为 HR，我希望能够管理员工合同到期提醒，以便及时续签
12. **US-012**：作为经理，我希望能够对下属进行绩效评分，以便量化团队表现
13. **US-013**：作为 HR，我希望能够查看组织架构图，以便了解公司人员分布
14. **US-014**：作为员工，我希望能够查看自己的假期余额，以便合理规划休假
15. **US-015**：作为 HR，我希望能够导出各类报表（Excel/PDF），以便汇报工作

### P2 期望故事
16. **US-016**：作为 HR，我希望能够设置培训课程并追踪完成情况，以便提升员工技能
17. **US-017**：作为经理，我希望能够查看团队人力成本分析，以便控制预算
18. **US-018**：作为系统管理员，我希望能够查看操作日志，以便审计追踪

## 三、需求池

### P0（必须有 — MVP 核心）

| 编号 | 模块 | 功能 | 描述 |
|------|------|------|------|
| F-001 | 用户与权限 | 登录/登出 | 邮箱+密码登录，JWT Token 鉴权 |
| F-002 | 用户与权限 | 角色管理 | 预设角色：超级管理员、HR、经理、员工；支持自定义权限 |
| F-003 | 员工管理 | 员工信息 CRUD | 姓名工号部门岗位入职日期联系方式等 |
| F-004 | 员工管理 | 员工列表 | 分页、搜索、筛选（部门/状态/入职日期） |
| F-005 | 员工管理 | 批量导入 | Excel 模板下载+上传解析导入 |
| F-006 | 考勤管理 | 在线打卡 | GPS 定位打卡，支持上下班打卡 |
| F-007 | 考勤管理 | 考勤记录 | 日/月考勤统计，异常标记（迟到/早退/缺勤） |
| F-008 | 请假管理 | 请假申请 | 选择假别+起止日期+事由，提交审批流 |
| F-009 | 请假管理 | 审批流程 | 经理审批 → HR 备案，支持通过/驳回+意见 |
| F-010 | 薪资管理 | 薪资计算 | 基本工资+岗位津贴+加班费-社保-个税 |
| F-011 | 薪资管理 | 薪资明细 | 员工查看个人薪资条，HR 查看全员汇总 |
| F-012 | 招聘管理 | 职位发布 | 发布/编辑/关闭招聘职位 |
| F-013 | 招聘管理 | 候选人管理 | 简历录入/导入，面试流程追踪 |
| F-014 | 仪表盘 | 数据概览 | 员工总数/本月入职离职/考勤异常/待审批 |

## 四、UI 设计要点

### 4.1 整体风格
- **设计语言**：企业级 SaaS 风格，简洁专业
- **配色方案**：主色 #1976D2（蓝），辅色 #4CAF50（绿），警示 #F44336（红）
- **布局**：左侧导航栏 + 顶部工具栏 + 右侧内容区

### 4.2 核心页面
1. **登录页**：品牌 Logo + 登录表单
2. **仪表盘**：数据卡片 + 图表（ECharts）
3. **员工列表**：表格 + 搜索栏 + 批量操作
4. **员工详情**：Tab 切换（基本信息/合同/考勤/薪资）
5. **考勤页面**：日历视图 + 统计数据
6. **薪资页面**：月度汇总表 + 薪资条详情
7. **招聘看板**：看板视图（待筛选/面试中/已发offer/已入职）

### 4.3 响应式
- 最小支持 1280px 宽度
- 关键页面支持移动端适配

## 五、非功能性需求

| 类别 | 需求 | 指标 |
|------|------|------|
| 性能 | 页面加载时间 | < 2s（首屏） |

| 数据 | 备份策略 | 每日自动备份 |
